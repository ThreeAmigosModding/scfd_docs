# Complete Police System (/docs/paid/police)





<ResourceBtns resource="complete-police-system" />

SCFD Police System is a fully modular, performance-optimized law enforcement framework for FiveM, designed to provide immersive policing mechanics while remaining flexible for any server style.

Built from the ground up with scalability and compatibility in mind, the system supports multiple departments, realistic investigation tools, AI-assisted utilities, and configurable policing workflows - all structured to work seamlessly together or independently.

Every feature is designed to integrate cleanly into existing server ecosystems without restriction, allowing servers to keep their preferred systems while upgrading others.

Feedback and feature ideas have been contributed by individuals who currently work, or have previously worked, in real-world law enforcement roles, helping shape systems that feel authentic, practical, and immersive for serious RP environments.

## Key Features: [#key-features]

* 🎥 Security Camera System: Monitor configurable camera networks across your server.
* 🚓 Police Vehicle Garage: Department-managed fleet system with vehicle tracking and return logic.
* 🔫 Advanced Police Armory: Configurable weapon and equipment distribution system.
* 👕 Locker & Clothing Rooms: Department-specific uniforms and locker functionality.
* 🎯 Police Actions Menu: Supports both radial and standard interaction menus.
* 🔗 Cuffing & Zipties: Multiple restraint types with animations and configurable behavior.
* 🔊 ShotSpotter Integration: Automatically detect and report nearby gunfire activity.
* 🙌 Hands Up System: Player surrender mechanic with animation support.
* 🕶️ Tint Meter Tool: Measure and enforce vehicle window tint limits.
* 🧬 Fingerprint Reader: Capture and analyze fingerprint evidence from suspects and scenes.
* 🚧 Placeable Police Objects: Deploy cones, barriers, spike strips and more.
* 📹 Live Body Cameras: Real-time officer bodycam functionality.
* 🧪 Evidence System: Collect and analyze casings, blood samples, GSR, BAC and fingerprints.
* 📍 Duty Blips: Identify active officers on the map based on department duty status.
* 🔍 Frisking & Searching: Immersive search mechanics with animations.
* ⛓️ Jailing Integrations: Compatible with existing jail resources.
* 🏢 Multi-Department Support: Configure multiple agencies with independent permissions and equipment.
* 🗄️ Department Facilities: Personal lockers and boss management menus.
* 📸 ALPR Integration: Automatically detect and flag vehicle license plates.
* 🚚 AI Tow Service: Request AI tow trucks with impound functionality.
* 🐕 K9 Unit System: Deploy police dogs for searches and suspect tracking.
* 🚨 Panic Button: Emergency distress alert system for officers.
* 🤼 Tackle Mechanic: Physically stop fleeing suspects.
* 💸 Fines System: Issue configurable monetary penalties.
* 🚦 Opticom Integration: Automatically control traffic lights during emergency response.

## Modular Design [#modular-design]

Each component is designed as an independent module, allowing servers to enable or disable features as needed without impacting performance or compatibility.

The system architecture prioritizes:

* Performance optimization
* Clean framework abstraction
* Config-driven behavior
* Expandability for future features
* Compatibility with existing server ecosystems

## Integration & Customization [#integration--customization]

* Compatible with all major frameworks (QB, Qbox, ND, ESX) or works standalone.
* Supports any\* inventory, target system, notifications, progressbars & skillchecks.
* Bridge system included with hooks for custom integrations and extensions.
* Fully configurable data structure allowing extensive customization.
* Localization support for any language using ox\_lib locales.

-# \* *Includes various integrations out of the box, with the ability to add your own via expansive hooks and bridge. Please see the documentation for all included*
